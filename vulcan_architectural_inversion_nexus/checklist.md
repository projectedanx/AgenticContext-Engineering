# Scar-Driven Topological Extruder (SDTE) - Strict Structural Checklist

## I. Epistemic Gates
- [ ] **Friction Ingestion Verified:** The system explicitly requires human input regarding failure modes (Scars) and contradictory NFRs prior to generating structural suggestions.
- [ ] **Paraconsistent Mapping Assured:** Contradictory inputs are maintained in the latent space without premature averaging or sycophantic resolution.

## II. Structural & Topological Invariants
- [ ] **Mereological Mandate Enforced:** Zero cross-domain state mutation calls exist in the generated DAG. Transitivity of data access across bounded contexts is blocked.
- [ ] **Shared Database Anathema Respected:** The system automatically halts and rejects any topological output proposing two distinct bounded contexts writing to the same database schema. (Integration must be event-driven).
- [ ] **Bricolage Lens Applied:** Unwarranted complexity (e.g., unnecessary microservices without justifying scale/deployment cadence differences) is rejected in favor of Modular Monolith recommendations.

## III. Extrusion Quality Control
- [ ] **DCCD Schema Compliance:** All generated outputs adhere strictly to predefined structural schemas (C4_Model_ADR_JSON, etc.).
- [ ] **Adjectival Bounds Met:** Generative output contains zero "evaluative marketing adjectives" (e.g., robust, scalable, seamless). Prose relies entirely on mathematical and structural definitions.
- [ ] **Epistemic Escrow Integrity:** The CFDI Brake is functional; if perfect consistency and availability under partition (CAP violation) are demanded, the system halts execution.
